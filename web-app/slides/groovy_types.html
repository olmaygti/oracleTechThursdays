Groovy types and literals

There are not primitive types in Groovy

  • Everything is an object
  • Numbers fall under the best fitter (Integer, Double, BigDecimal...)
  • boolean literals are instance of the Boolean class
  • Special inline syntax for collections
    • [] for ArrayList
    • [:] for HashMap
  • Strings are special ...

Groovy Strings literals

There are multiple ways to declare a String literal in Groovy

  • Single quoted: 'this is a String'
  • Double quoted: "this is a String, but not always"
  • Slashy Strings: /This is really handy for regexps!/
  • Dollar-Slashy Strings: $/Really?/$
  • You can declare multiline Strings with triple single/double quotes
